# Hyperfocus 스펙

## 1. 개요

### 1.1 제품 정의

macOS 상단 메뉴바에 상주하는 스탑워치 앱. 사용자는 메뉴바에서 즉시 작업 시간을 시작/정지하고,
하루 동안 어떤 작업에 얼마만큼의 시간을 썼는지를 기록한다.

### 1.2 사용 목적

- 지금 진행 중인 태스크에 몇 시간을 썼는지 즉시 확인한다.
- 하루 동안 누적된 총 작업 시간을 확인한다.
- 하루가 끝나면 그 하루 안에서 어떤 작업에 얼마만큼 시간을 썼는지 통계로 본다.
- 여러 날을 누적하여 주간 평균 작업 시간을 파악한다.

### 1.3 사용 환경

- 단일 사용자, 단일 기기. 동기화 없음.
- 모든 데이터는 사용자의 기기에 로컬로만 보관된다.

## 2. 정의

### 2.1 용어 정의

이 문서 전반에서 사용하는 핵심 용어. UI 라벨은 이와 동일하게 표기한다.

- **세션 (Session)**
  하나의 작업 단위. 사용자가 "지금 어떤 일을 하고 있는지"를 나타낸다.
  Title(선택)과 누적 시간을 가진다.
- **하루 (Day)**
  "종료"와 "종료" 사이의 기간. 새벽 6시부터 다음 날 새벽 6시까지를 하루로 정의한다.
  하나의 하루는 0개 이상의 세션을 포함한다.
- **현재 세션 시간 (Current)**
  지금 진행 중인 세션에서 누적된 시간.
- **전체 시간 (Today)**
  오늘 하루에 속한 모든 세션의 누적 시간(현재 진행 중 세션 포함)의 합.
- **세션 리셋 (Reset)**
  현재 세션을 종료(저장)하고 타이머를 idle 상태(세션 없음, 정지)로 되돌리는 행위.
- **종료 (End)**
  현재 하루를 마감하고 새로운 빈 하루를 시작하는 행위.

### 2.2 상태 정의

타이머의 상태.

- **idle**: `activeSession == nil, isRunning == false` → `Start` 버튼
- **running**: `activeSession != nil, isRunning == true` → `Pause` + `Reset` 버튼
- **paused**: `activeSession != nil, isRunning == false` → `Resume` + `Reset` 버튼

## 3. 메뉴바(상단바) 동작

### 3.1 메뉴바 라벨

**항상 `HH:MM:SS` 텍스트**로 표시한다. 아이콘으로 전환되지 않는다.

- 표시 형식: `HH:MM:SS` (예: `01:23:45`).
- idle 상태에서도 `00:00:00`으로 표시한다.
- running/paused 여부와 무관하게 항상 현재 세션 누적 시간을 텍스트로 유지한다.
- 텍스트는 매 1초마다 갱신된다.

### 3.2 메뉴바 상호작용

- 라벨을 클릭하면 **Popover**가 메뉴바 바로 아래에 표시된다.
- Popover가 열린 상태에서 라벨을 다시 클릭하거나 Popover 바깥을 클릭하면 Popover가 닫힌다.
- Popover가 닫혀 있어도 스탑워치 동작은 계속된다(메뉴바 라벨도 갱신).
- Popover가 열린 상태에서 **스페이스(Space) 키**를 누르면 타이머 상태에 따라 다르게 동작한다.
  - **running** 상태이면: Pause하고 Popover는 닫히지 않는다.
  - **paused** 상태이면: Resume하고 Popover가 닫힌다.
  - **idle** 상태이거나 Title 입력 필드에 포커스가 있는 경우: 단축키가 동작하지 않는다(필드 안에서 Space는 공백 입력으로 처리).
  - 이 단축키는 Popover가 열려 있는 동안만 동작하며 시스템 전역 단축키(§11)가 아니다.

### 3.3 Dock / 앱 스위처

- 이 앱은 Dock에 보이지 않는다.
- Cmd+Tab 앱 스위처에도 나타나지 않는다.
- 별도의 메인 윈도우를 띄우지 않는다(메뉴바 + Popover로만 동작).

## 4. Popover 구성

Popover에는 두 개의 탭(또는 전환 가능한 두 화면)이 있다: **타이머 화면**과 **통계 화면**.
기본 진입 화면은 **타이머 화면**이다.

### 4.1 타이머 화면

위에서 아래로 다음 순서로 구성한다.

1. **Title 필드**
   - Title이 비어 있을 때: 한 줄짜리 텍스트 입력 필드. placeholder는 `Title`.
   - Title이 입력된 상태일 때: 제목 텍스트와 그 오른쪽에 편집 아이콘을 함께 표시 (입력 필드 아님). 편집 아이콘을 누르면 입력 필드로 전환된다.
   - 입력은 언제든 수정 가능하며, 입력 즉시 현재 세션의 Title로 저장된다.
   - Popover가 열릴 때 **idle 상태이면** 이 필드에 자동으로 포커스가 이동한다(running/paused이면 자동 포커스 없음).
   - idle 상태에서 이 필드에 포커스가 있는 채로 **엔터(Return) 키**를 누르면 세션이 시작되고 Popover가 닫힌다(Title이 비어 있어도 동작).

2. **시간 표시**
   - 두 줄을 동일한 폰트 크기로 나란히 표시한다.
   - 단, Current와 Today 라벨은 회색 폰트를 사용한다.
     ```
     Current   HH:MM:SS
     Today     HH:MM:SS
     ```
   - `Current`는 현재 세션 시간, `Today`는 오늘 하루의 전체 누적 시간.
   - 매초 갱신.

3. **기본 컨트롤 버튼** — 위에 구분선(divider)이 있으며, 타이머 상태에 따라 달라진다.
   - **idle**: `Start` 버튼 1개 (파란색, 가운데).
   - **running**: `Pause`(파란색, 좌) + `Reset`(우) 2개 나란히.
   - **paused**: `Resume`(파란색, 좌) + `Reset`(우) 2개 나란히.

4. **보조 컨트롤 버튼** — 위에 구분선(divider)이 있으며, 타이머 상태와 무관하게 항상 표시된다.
   - `End`(좌) + `Stats`(우) 2개 나란히.
   - `End`: 현재 하루를 마감하는 전체 리셋.
   - `Stats`: 통계 화면으로 전환.

### 4.2 통계 화면

위에서 아래로 다음 순서로 구성한다.

1. **요약 영역**
   - 최근 7일의 평균 작업 시간(일평균).
   - 표시 형식: `5h 42m` 같은 사람 친화적 형식.
   - 기록된 날이 7일 미만이면 있는 만큼만 평균(개수도 함께 표기, 예: "최근 3일 평균 4h 12m").
2. **오늘 카드**
   - 항상 가장 위(요약 다음)에 위치.
   - 시작 일시 (예: `2026-05-25 09:14 시작`).
   - 누적 시간 합계.
   - 같은 Title끼리 합산된 세션 목록 (섹션 7.2 참고).
3. **지난 날 목록**
   - 최근 날이 위에 오도록 시간 역순.
   - 각 항목 요약: 하루 시작 일시, 종료 일시, 총 누적 시간.
   - 항목을 클릭하면 그 날의 상세 화면(섹션 4.3)으로 이동한다.

### 4.3 하루 상세 화면

- 진입 경로: 통계 화면의 지난 날 항목 클릭.
- 표시 항목:
  - 하루 시작/종료 일시.
  - 하루 총 누적 시간.
  - 같은 Title끼리 합산된 세션 목록 (Title, 합산 시간). 시간 내림차순 정렬.
- 이전 화면(통계)으로 돌아갈 수 있는 명확한 진입점(뒤로 가기 버튼 또는 동등한 UI).

## 5. 타이머 동작 상세

### 5.1 시작 (`Start` 버튼 또는 Return 키)

- 현재 진행 중인 세션이 없으면 새 세션을 만든다(Title은 빈 문자열로 시작, 시작 시각은 현재).
- 타이머를 동작 상태로 전환한다.
- 매초마다 **Current**와 **Today**가 동일하게 1초씩 증가한다.
- idle 상태에서 Title 입력 필드에 포커스가 있는 채로 **엔터(Return) 키**를 눌러도 동일하게 시작된다(이 경우 Popover도 닫힘, §4.1 참고).

### 5.2 일시정지 (`Pause` 버튼)

- 타이머를 정지 상태로 전환한다.
- 현재 세션과 그 누적 시간은 그대로 유지된다(다음 `Resume` 시 이어서 누적된다).
- Title도 그대로 유지된다.
- 버튼이 `Resume` + `Reset` 조합으로 바뀐다.

### 5.3 세션 리셋 (`Reset` 버튼)

- 타이머가 실행 중이었다면 먼저 정지한다.
- 현재 세션을 종료한다.
  - 종료된 세션의 누적 시간이 0보다 크면 **오늘 하루의 세션 기록**에 추가된다.
  - 누적 시간이 0이면(시작 직후 바로 리셋한 경우 등) 기록하지 않는다.
- **idle 상태로 복귀한다.** 진행 중 세션 없음, 타이머 정지.
- **Title 필드가 비워지고 입력 모드로 복귀한다.**
- 버튼이 `Start` 1개로 바뀐다.
- Today 표시는 오늘 하루에서 마감된 세션들의 누적 합계로 유지된다(새 진행 중 세션 없음).
- 타이머가 실행 중이든 일시정지 중이든 동일하게 idle로 복귀한다.

### 5.4 종료 (`End` 버튼)

1. 진행 중이던 세션이 있고 누적 시간이 0보다 크면 그 세션을 오늘 하루의 세션 기록에 추가한다.
2. 오늘 하루를 종료된 상태(종료 시각 = 현재)로 마감하고 지난 날 목록에 보관한다.
3. 새로운 하루를 시작한다.
4. 진행 중 세션 없음, 타이머 정지 상태로 만든다.
5. 메뉴바 라벨 및 화면의 Current와 Today는 `00:00:00`으로 표시된다.
6. Title 필드가 비워지고 입력 모드로 복귀한다.

### 5.5 새벽 6시 자동 마감

- 매일 새벽 6시 0분 0초에 현재 하루를 자동으로 마감하고 새로운 하루를 시작한다.
- 자동 마감 시 동작 순서:
  1. 타이머가 실행 중이었다면 자동으로 일시정지한다.
  2. 진행 중이던 세션이 있고 누적 시간이 0보다 크면 그 세션을 마감 하루의 기록에 추가한다.
  3. 현재 하루를 종료된 상태(종료 시각 = 새벽 6시)로 마감하고 지난 날 목록에 보관한다.
     세션이 하나도 없거나 모두 누적 시간 0인 경우, 빈 하루는 목록에 추가하지 않는다.
  4. 새로운 하루를 시작한다.
  5. idle 상태로 전환된다. Title 필드가 비워지고 입력 모드로 복귀한다.
- 앱이 새벽 6시에 실행 중이지 않았더라도, 다음 번 앱 실행(또는 슬립에서 깨어남) 시 마지막으로 저장된 하루가 현재 하루와 다르면(즉 새벽 6시를 넘겼으면) 자동 마감을 수행한다.
  - 이때 종료 시각은 실제 새벽 6시로 기록하고, 진행 중이던 세션의 누적 시간은 그대로 보존하여 마감 하루에 포함한다.
- 여러 번의 새벽 6시를 건너뛴 경우(며칠째 앱을 켜지 않은 경우), 마지막 저장 상태를 한 번만 마감하고 새로운 하루로 진입한다(중간의 빈 날은 생성하지 않는다).

## 6. Title 규칙

### 6.1 입력

- Title 입력은 항상 선택사항이다. 비어 있어도 Start / Pause / Resume / Reset / End 모두 가능하다.
- 진행 중 세션의 Title은 언제든 수정 가능하다(시작 전, 중, 일시정지 중 모두).
- 입력은 한 줄, 자유 텍스트. 길이 제한은 두지 않지만 통계 화면에서는 너무 길면 잘려 보일 수 있다.

### 6.2 저장 표기

- Title이 빈 채로 세션이 종료되어 기록될 때, 저장 및 통계 표시상의 Title은 `(Untitled)`로 한다.
- 좌우 공백만 입력된 경우(`"   "`)도 빈 Title로 간주하여 `(Untitled)`로 저장한다.

### 6.3 합산 규칙

- 통계에서 "같은 Title"이란 좌우 공백 제거 후의 문자열이 정확히 일치하는 것을 말한다.
- 대소문자는 구분한다 (`React` 와 `react`는 다른 Title로 본다).
- `(Untitled)`로 표기되는 세션들도 자기들끼리 하나로 합산된다.

## 7. 통계 규칙

### 7.1 무엇이 기록되는가

- 각 하루에는 그 하루 안에서 종료된 모든 세션이 시간 순서대로 기록된다.
- 세션 한 건당 기록되는 정보: Title, 누적 시간, 시작 시각.
- 누적 시간이 0인 세션(시작도 안 했거나, 시작 직후 즉시 리셋된 경우)은 기록하지 않는다.

### 7.2 표시 방식 — 합산 뷰

- 하루를 통계 화면에서 보여줄 때, 같은 Title 세션들은 하나의 항목으로 합산해서 보여준다.
- 합산 후 시간 내림차순으로 정렬한다.
- 예시:
  ```
  리액트 공부      2h 30m
  코드 리뷰        1h 05m
  (Untitled)      0h 45m
  ─────────────────────
  합계            4h 20m
  ```

### 7.3 일평균 계산

- 통계 화면 상단 요약의 "평균"은 다음으로 계산한다.
  - 대상: 오늘을 제외한, 마감된 최근 7일.
  - 평균 = (대상 날들의 총 누적 시간 합) ÷ (대상 날 수).
- 과거 기록이 없으면 요약 영역에 `아직 통계가 없습니다.` 같은 안내를 표시한다.
- 기록된 날이 7일 미만이면 평균값과 함께 실제 사용된 날 수를 명시한다.

## 8. 슬립 및 잠금 처리

### 8.1 자동 일시정지

- 다음 시점에 스탑워치가 실행 중이라면 자동으로 일시정지 상태로 전환된다.
  - 시스템이 슬립(잠자기) 모드로 진입하는 시점.
- 일시정지 시점의 Current와 Today는 그대로 보존된다.
- 자동 일시정지로 인해 손실되는 시간이 통계에 추가되지 않는다(자리 비운 시간을 작업 시간으로 치지 않는다).

### 8.2 자동 재개하지 않음

- 시스템이 깨어난 후 스탑워치를 **자동으로 다시 시작하지 않는다.**
- 사용자가 직접 Popover를 열고 `Resume` 버튼을 눌러야 재개된다.
  (슬립으로 인한 자동 일시정지는 항상 activeSession이 존재하는 paused 상태이므로 `Resume`이 표시됨.)

## 9. 데이터 보존 정책

- 모든 데이터는 사용자의 기기에 로컬로만 저장된다. 네트워크 전송 없음.
- 앱을 종료했다가 다시 열어도 다음 상태가 복원된다.
  - 오늘 하루와 그 하루 안에서 마감된 세션들.
  - 진행 중이던 세션의 Title과 누적 시간.
  - 종료 시점에 타이머가 실행 중이었더라도 복원 시에는 **항상 정지 상태로 시작한다**(슬립 처리와 같은 원칙: 자리 비운 시간을 가산하지 않는다).
    - 진행 중이던 세션이 있으면 paused 상태(`Resume` + `Reset` 버튼).
    - 진행 중이던 세션이 없으면 idle 상태(`Start` 버튼).
- 과거 날 데이터는 사용자가 명시적으로 지우기 전까지 무기한 보관된다.
  (보관 기간 자동 제한 없음.)

## 10. 엣지 케이스 정리

| 상황                                                                      | 기대 동작                                                                                                                 |
| ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `Start` 직후 곧바로 `Reset`                                               | 누적 시간 0인 세션이라 기록되지 않음. idle 상태로 복귀 (`Start` 버튼, Title 필드 초기화).                                 |
| `Pause` 중 `Reset`                                                        | 진행 중 세션을 종료(duration > 0이면 저장)하고 idle 상태로 복귀. `Start` 버튼 표시.                                       |
| `Start` 직후 곧바로 `End`                                                 | 누적 0 세션은 기록되지 않은 채 현재 하루 마감. 새 하루로 진입.                                                            |
| 진행 중 세션 없이 `End` (이미 정지 상태)                                  | 현재 하루를 그대로 마감(누적 0이면 빈 하루 그대로 종료) 후 새 하루 생성.                                                  |
| 진행 중 세션이 한 번도 시작되지 않은 빈 하루에서 `End`                    | 빈 하루는 마감해도 의미가 없으므로 지난 날 목록에 추가하지 않는다. 사용자 입장에서 아무 일도 일어나지 않은 것처럼 보인다. |
| 슬립 중 새벽 6시를 넘김                                                   | 슬립에서 깨어날 때 자동 마감 수행(§5.5). 진행 중이던 세션은 마감 하루에 저장되고, 새 하루 idle 상태로 진입한다.           |
| 앱 종료 중 새벽 6시를 넘김                                                | 다음 앱 실행 시 자동 마감 수행(§5.5). 마지막 저장 세션은 마감 하루에 포함되고, 새 하루 idle 상태로 시작한다.              |
| 며칠째 앱을 켜지 않아 새벽 6시를 여러 번 넘김                             | 마지막 저장 상태를 한 번만 마감. 중간의 빈 날은 생성하지 않는다. 새 하루 idle 상태로 시작.                                |
| 슬립 중에 사용자가 아무것도 하지 않고 다시 깨어남                         | 그대로 paused 상태로 머문다. 사용자가 직접 `Resume`을 눌러야 재개.                                                        |
| Title이 매우 긴 경우                                                      | 입력은 허용. 표시 모드(label)와 통계 화면에서는 한 줄 안에서 ellipsis로 잘려 보일 수 있다. 합산은 잘리기 전 원본 기준.    |
| 같은 Title의 세션을 한 하루에 여러 번 한 경우                             | 통계 합산 뷰에서 하나로 합쳐서 표시.                                                                                      |
| 앱 종료 직전에 마지막 1초가량                                             | 1초 미만의 미세한 손실은 허용한다.                                                                                        |
| Popover 열린 상태, running 중 Space 키 (Title 필드 포커스 없음)           | Pause로 전환. Popover는 닫히지 않고 유지.                                                                                 |
| Popover 열린 상태, paused 중 Space 키 (Title 필드 포커스 없음)            | Resume으로 전환 후 Popover가 닫힌다.                                                                                      |
| Popover가 idle 상태로 열림                                                | Title 입력 필드에 자동 포커스.                                                                                            |
| idle, Title 입력 필드 포커스, Return 키                                   | 세션 시작 후 Popover 닫힘 (Title 없어도 동작).                                                                            |
| idle, Title 입력 필드 포커스, Space 키                                    | 타이머 제어 없이 필드에 공백 문자만 입력됨.                                                                               |
| Popover가 running/paused 상태로 열림                                      | Title 필드에 자동 포커스 없음.                                                                                            |
| running/paused 상태, 편집 아이콘 클릭 후 Title 입력 필드 포커스, Space 키 | 타이머 제어 없이 필드에 공백 문자만 입력됨.                                                                               |
| Title이 입력된 상태에서 편집 아이콘 클릭                                  | 동일한 위치가 입력 필드로 전환된다. 기존 Title 텍스트는 그대로 유지된다.                                                  |

## 11. 비-목표 (Out of Scope)

이번 버전에서는 다음 기능들을 다루지 않는다.

- 여러 기기 간 동기화, 클라우드 백업.
- 글로벌 키보드 단축키.
- macOS 로그인 시 자동 실행 옵션(사용자가 직접 실행한다).
- 일/주/월 단위 차트나 그래프 시각화(텍스트 기반 목록 + 평균값만 제공).
- 알림(작업 시간 도달, 휴식 권장 등) 기능.
- 작업 카테고리/태그/색상 등의 분류 체계(Title만으로 식별).
- 데이터 내보내기/가져오기, CSV 등 외부 형식 지원.
- 동시에 여러 세션을 병렬로 돌리는 기능.
