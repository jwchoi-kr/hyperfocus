# Hyperfocus 스펙

## 1. 개요

### 1.1 제품 정의

macOS 상단 메뉴바에 상주하는 스탑워치 앱. 사용자는 메뉴바에서 즉시 작업 시간을 시작/정지하고,
하루 동안 어떤 작업에 얼마만큼의 시간을 썼는지를 기록한다.

### 1.2 사용 목적

- 지금 진행 중인 태스크에 몇 시간을 썼는지 즉시 확인한다.
- 하루 동안 누적된 총 작업 시간을 확인한다.
- 하루가 끝나면 그 하루 안에서 어떤 작업에 얼마만큼 시간을 썼는지 통계로 본다.
- 여러 날을 누적하여 주간 평균 작업 시간을 파악한다.

### 1.3 사용 환경

- 단일 사용자, 단일 기기. 동기화 없음.
- 모든 데이터는 사용자의 기기에 로컬로만 보관된다.

## 2. 정의

### 2.1 용어 정의

이 문서 전반에서 사용하는 핵심 용어. UI 라벨은 이와 동일하게 표기한다.

- **세션 (Session)**
  하나의 작업 단위. 사용자가 "지금 어떤 일을 하고 있는지"를 나타낸다.
  Title(선택)과 누적 시간을 가진다.
- **하루 (Day)**
  "종료"와 "종료" 사이의 기간. 새벽 6시부터 다음 날 새벽 6시까지를 하루로 정의한다.
  하나의 하루는 0개 이상의 세션을 포함한다.
- **현재 세션 시간 (Current)**
  지금 진행 중인 세션에서 누적된 시간.
- **전체 시간 (Today)**
  오늘 하루에 속한 모든 세션의 누적 시간(현재 진행 중 세션 포함)의 합.
- **세션 종료 (End)**
  현재 세션을 종료(저장)하고 타이머를 idle 상태(세션 없음, 정지)로 되돌리는 행위. UI 버튼 라벨은 `End`.

### 2.2 상태 정의

타이머의 상태.

- **idle**: `activeSession == nil, isRunning == false` → `Start` 버튼
- **running**: `activeSession != nil, isRunning == true` → `Pause`(회색) + `End`(빨간색) 버튼
- **paused**: `activeSession != nil, isRunning == false` → `Resume`(파란색) + `End`(빨간색) 버튼

## 3. 메뉴바(상단바) 동작

### 3.1 메뉴바 라벨

아이콘으로 전환되지 않는다. **항상 이름(위) + 타이머(아래) 두 줄**로 표시한다.

```
리액트 공부
03:10:11
```

- **이름 줄:** 진행 중 세션의 Title을 표시한다. Title이 비어 있거나 공백만 있으면 `(Untitled)`로 표시한다. idle 상태(세션 없음)도 `(Untitled)`로 표시한다.
- **타이머 줄:** 현재 세션 누적 시간을 `HH:MM:SS` 형식으로 표시한다. idle 상태에서는 `00:00:00`.
- 이름이 너무 길면 타이머(`HH:MM:SS`) 텍스트 너비를 기준으로 `…`로 말줄임한다. 메뉴바 아이템 너비는 항상 타이머 너비+여백으로 고정되어 이름 길이에 따라 변하지 않는다. 합산 등 데이터 처리는 잘리기 전 원본 기준.
- running/paused 여부와 무관하게 타이머는 항상 현재 세션 누적 시간을 유지한다.
- 타이머 텍스트는 매 1초마다 갱신된다.

### 3.2 메뉴바 상호작용

- 라벨을 클릭하면 **Popover**가 메뉴바 바로 아래에 표시된다.
- Popover가 열린 상태에서 라벨을 다시 클릭하거나 Popover 바깥을 클릭하면 Popover가 닫힌다.
- Popover가 닫혀 있어도 스탑워치 동작은 계속된다(메뉴바 라벨도 갱신).
- Popover가 열린 상태에서 **스페이스(Space) 키**를 누르면 타이머 상태에 따라 다르게 동작한다.
  - **running** 상태이면: Pause하고 Popover는 닫히지 않는다.
  - **paused** 상태이면: Resume하고 Popover가 닫힌다.
  - **idle** 상태이거나 Title 입력 필드에 포커스가 있는 경우: 단축키가 동작하지 않는다(필드 안에서 Space는 공백 입력으로 처리).
  - 이 단축키는 Popover가 열려 있는 동안만 동작하며 시스템 전역 단축키(§11)가 아니다.

### 3.3 Dock / 앱 스위처

- 이 앱은 Dock에 보이지 않는다.
- Cmd+Tab 앱 스위처에도 나타나지 않는다.
- 별도의 메인 윈도우를 띄우지 않는다(메뉴바 + Popover로만 동작).

## 4. Popover 구성

Popover에는 두 개의 탭(또는 전환 가능한 두 화면)이 있다: **타이머 화면**과 **통계 화면**.
기본 진입 화면은 **타이머 화면**이다.

Popover 크기는 화면에 따라 다르다. **타이머 화면**은 너비 300pt(높이 자동), **통계·상세 화면**은 **420 × 820pt**로 고정한다.

### 4.1 타이머 화면

위에서 아래로 다음 순서로 구성한다.

1. **Title 필드**
   - Title이 비어 있을 때: 한 줄짜리 텍스트 입력 필드. placeholder는 `Title`.
   - Title이 입력된 상태일 때: 제목 텍스트와 그 오른쪽에 편집 아이콘을 함께 표시 (입력 필드 아님). 편집 아이콘을 누르면 입력 필드로 전환된다.
   - 입력은 언제든 수정 가능하며, 입력 즉시 현재 세션의 Title로 저장된다.
   - Popover가 열릴 때 **idle 상태이면** 이 필드에 자동으로 포커스가 이동한다(running/paused이면 자동 포커스 없음).
   - idle 상태에서 이 필드에 포커스가 있는 채로 **엔터(Return) 키**를 누르면 세션이 시작되고 Popover가 닫힌다(Title이 비어 있어도 동작).

2. **시간 표시**
   - 두 줄을 동일한 폰트 크기로 나란히 표시한다.
   - 단, Current와 Today 라벨은 회색 폰트를 사용한다.
     ```
     Current   HH:MM:SS
     Today     HH:MM:SS
     ```
   - `Current`는 현재 세션 시간, `Today`는 오늘 하루의 전체 누적 시간.
   - 매초 갱신.

3. **기본 컨트롤 버튼** — 위에 구분선(divider)이 있으며, 타이머 상태에 따라 달라진다.
   - **idle**: `Start` 버튼 1개 (파란색, 가운데).
   - **running**: `Pause`(회색, 좌) + `End`(빨간색, 우) 2개 나란히.
   - **paused**: `Resume`(파란색, 좌) + `End`(빨간색, 우) 2개 나란히.

4. **보조 컨트롤 버튼** — 위에 구분선(divider)이 있으며, 타이머 상태와 무관하게 항상 표시된다.
   - `Stats` 버튼(좌) + `Focus` 버튼(우) 두 개 나란히.
   - `Stats`: 통계 화면으로 전환.
   - `Focus`: Focus Mode 설정 화면으로 전환.

### 4.2 통계 화면

위에서 아래로 다음 순서로 구성한다.

1. **Today 섹션(통계)**
   - 항상 가장 위에 위치.
   - 날짜 헤더 (예: `Tue, May 26`), 총 누적 시간. 우상단에 시작/종료 시각 표시.
   - 세션 목록 (시작 시각 오름차순 정렬). 세션이 없으면 목록 없이 날짜와 총 시간만 표시.
   - 각 세션 항목에는 오늘 전체 시간 대비 백분율을 표시한다. 시간 텍스트(`1h 05m` 등) 왼쪽에 `32%` 형식으로 표시하며, 모든 세션의 % 텍스트는 동일한 열(column)에 정렬된다.
   - **세션 편집**: 각 세션 항목의 이름 왼쪽에 편집(✎) 아이콘과 삭제(🗑) 아이콘이 나란히 표시된다.
     - 편집 아이콘 클릭 → 이름 부분이 인라인 입력 필드로 전환된다. Return(엔터) 키를 누르면 저장하고 표시 모드로 복귀한다. 빈 문자열이나 공백만 입력하면 `(Untitled)`로 저장된다. 해당 세션의 이름이 변경된다.
     - 삭제 아이콘 클릭 → `[Delete]` / `[Cancel]` 두 버튼으로 구성된 확인 UI가 표시된다(텍스트 메시지 없음, 버튼 왼쪽 정렬). 확인(`[Delete]`) 시 해당 세션이 삭제된다. Today 섹션(통계)은 세션을 모두 삭제해도 사라지지 않는다.
     - 진행 중인 세션(`activeSession`)은 편집/삭제 대상에 포함되지 않는다. Today 섹션(통계) 기준 편집 대상은 이미 종료·저장된 세션들이다.
2. **This Week 섹션**
   - Today 섹션 아래에 위치.
   - 이번 주(월~일) 날짜별 바 차트 + 최근 N일 평균 작업 시간 표시.
3. **This Month 섹션**
   - This Week 섹션 아래에 위치.
   - 섹션 헤더 라벨: `"This Month"`.
   - 이번 달의 날짜별 월간 캘린더를 표시한다. 캘린더 상단에는 `<` / `>` 버튼으로 이전·다음 달로 이동할 수 있으며, 월 타이틀(예: `May 2026`)을 탭하면 현재 달(오늘이 속한 달)로 이동한다. 캘린더 내 날짜 셀을 탭하면 해당 날의 상세 화면(§4.4)으로 이동한다.

### 4.3 Focus Mode 설정 화면

- 진입 경로: 타이머 화면 하단 `Focus` 버튼 클릭.
- **네비게이션 바 타이틀**: `Focus`. **뒤로 가기 버튼** 라벨: `Timer`.
- **차단 상태 배지**: 세션이 running 중이면 녹색 `Blocking Active`, 아니면 회색 `Inactive`.
- **macOS Focus 연동 섹션**: `macOS Focus 연동` 라벨과 토글 스위치로 구성.
  - 토글이 켜진 상태에서 세션이 running 으로 전환되면 macOS Shortcuts 앱의 `Focus On` 단축어를 즉시 실행한다.
  - 토글이 켜진 상태에서 세션이 running 에서 벗어나면(Pause, End, 슬립, 새벽 6시 마감, 앱 종료) `Focus Off` 단축어를 즉시 실행한다.
  - 세션 running 중 토글을 켜면 즉시 `Focus On` 실행, 토글을 끄면 즉시 `Focus Off` 실행.
  - 단축어 실행 실패(Shortcuts 미설치, 단축어 없음 등)는 에러를 무시하고 로그만 기록한다.
- **차단 앱 섹션**: 차단할 앱 목록. 각 행에 `×` 버튼으로 삭제. `+` 버튼 클릭 시 현재 실행 중인 앱 목록(AppPicker)을 표시하여 선택 추가.
- **차단 사이트 섹션**: 차단할 도메인 목록. 각 행에 `×` 버튼으로 삭제. `사이트 추가` 버튼 클릭 시 인라인 텍스트 필드 표시; Return 키로 도메인 저장.
- **세션 running 중에는 추가/삭제 버튼이 모두 비활성화된다.** (macOS Focus 연동 토글은 세션 running 중에도 변경 가능하다.)
- 설정은 앱 재시작 후에도 영구 보존된다(`state.json`에 저장).
- Popover 너비: 타이머 화면과 동일한 300pt.

#### 4.3.1 AppPicker

- 현재 실행 중인 앱 중 `activationPolicy == .regular`인 앱(Dock에 나타나는 일반 앱)만 표시.
- 이미 차단 목록에 있는 앱은 제외한다.
- 각 행에 앱 아이콘 + 이름 표시. 선택 즉시 차단 목록에 추가된다.
- 이름 오름차순으로 정렬.

#### 4.3.2 Focus Mode 동작

세션 running 중일 때만 차단이 활성된다. Pause, idle, 앱 종료 상태에서는 비활성.

| 이벤트 | 앱/사이트 차단 상태 변화 | macOS Focus 연동 (토글 켜짐 시) |
|--------|--------------------------|----------------------------------|
| `Start` / `Resume` | 즉시 활성화 | `Focus On` 단축어 실행 |
| `Pause` | 비활성화 | `Focus Off` 단축어 실행 |
| `End` | 비활성화 | `Focus Off` 단축어 실행 |
| 새벽 6시 자동 마감 | 비활성화 | `Focus Off` 단축어 실행 |
| 슬립 자동 일시정지 | 비활성화 | `Focus Off` 단축어 실행 |
| 앱 종료 | 비활성화 | `Focus Off` 단축어 실행 |

- **앱 차단**: `NSWorkspace.didLaunchApplicationNotification`으로 실행을 즉시 감지하여 강제 종료. 활성화 시점에 이미 실행 중인 차단 앱도 즉시 종료.
- **사이트 차단 (Chrome 전용)**: 2초 간격으로 Chrome 탭 URL을 조회하여 차단 도메인과 일치하는 탭을 닫음. Chrome이 실행 중이지 않으면 아무 동작도 하지 않음. 반응 지연 최대 2초.
- Chrome 자동화 권한 없음: macOS가 최초 실행 시 권한 요청 다이얼로그를 자동 표시. 거부 시 사이트 차단만 작동 안 함; 앱 차단은 정상 동작.

### 4.4 하루 상세 화면

- 진입 경로: This Month 섹션의 캘린더 날짜 셀 클릭.
- **네비게이션 바 타이틀**: 해당 날의 날짜를 `yyyy-MM-dd` 형식(예: `2026-05-04`)으로 표시한다.
- **뒤로 가기 버튼** 라벨: `Stats`.
- 표시 항목: Today 카드와 동일한 `DayCard` 레이아웃.
  - 날짜 헤더(`EEE, MMM d`), 총 누적 시간, 우상단에 시작/종료 시각.
  - 세션 목록 (시작 시각 오름차순 정렬).
- **세션 편집**: 각 세션 항목의 이름 왼쪽에 편집(✎) 아이콘과 삭제(🗑) 아이콘이 나란히 표시된다.
  - 편집 아이콘 클릭 → 이름 부분이 인라인 입력 필드로 전환된다. Return(엔터) 키를 누르면 저장하고 표시 모드로 복귀한다. 빈 문자열이나 공백만 입력하면 `(Untitled)`로 저장된다. 해당 세션의 이름이 변경된다.
  - 삭제 아이콘 클릭 → `[Delete]` / `[Cancel]` 두 버튼으로 구성된 확인 UI가 표시된다(텍스트 메시지 없음, 버튼 왼쪽 정렬). 확인(`[Delete]`) 시 해당 세션이 삭제된다. 그 결과 하루에 세션이 하나도 남지 않으면 하루 자체도 목록에서 제거되며 자동으로 통계 화면으로 이동한다.

## 5. 타이머 동작 상세

### 5.1 시작 (`Start` 버튼 또는 Return 키)

- 현재 진행 중인 세션이 없으면 새 세션을 만든다(Title은 빈 문자열로 시작, 시작 시각은 현재).
- 타이머를 동작 상태로 전환한다.
- 매초마다 **Current**와 **Today**가 동일하게 1초씩 증가한다.
- idle 상태에서 Title 입력 필드에 포커스가 있는 채로 **엔터(Return) 키**를 눌러도 동일하게 시작된다(이 경우 Popover도 닫힘, §4.1 참고).

### 5.2 일시정지 (`Pause` 버튼)

- 타이머를 정지 상태로 전환한다.
- 현재 세션과 그 누적 시간은 그대로 유지된다(다음 `Resume` 시 이어서 누적된다).
- Title도 그대로 유지된다.
- 버튼이 `Resume` + `End` 조합으로 바뀐다.

### 5.3 세션 종료 (`End` 버튼)

- 타이머가 실행 중이었다면 먼저 정지한다.
- 현재 세션을 종료한다.
  - 종료된 세션의 누적 시간이 0보다 크면 **오늘 하루의 세션 기록**에 추가된다.
  - 누적 시간이 0이면(시작 직후 바로 리셋한 경우 등) 기록하지 않는다.
- **idle 상태로 복귀한다.** 진행 중 세션 없음, 타이머 정지.
- **Title 필드가 비워지고 입력 모드로 복귀한다.**
- 버튼이 `Start` 1개로 바뀐다.
- Today(타이머 라벨) 표시는 오늘 하루에서 마감된 세션들의 누적 합계로 유지된다(새 진행 중 세션 없음).
- 타이머가 실행 중이든 일시정지 중이든 동일하게 idle로 복귀한다.

### 5.4 새벽 6시 자동 마감

- 매일 새벽 6시 0분 0초에 현재 하루를 자동으로 마감하고 새로운 하루를 시작한다.
- 자동 마감 시 동작 순서:
  1. 타이머가 실행 중이었다면 자동으로 일시정지한다.
  2. 진행 중이던 세션이 있고 누적 시간이 0보다 크면 그 세션을 마감 하루의 기록에 추가한다.
  3. 현재 하루를 종료된 상태(종료 시각 = 새벽 6시)로 마감하고 지난 날 목록에 보관한다.
     세션이 하나도 없거나 모두 누적 시간 0인 경우, 빈 하루는 목록에 추가하지 않는다.
  4. 새로운 하루를 시작한다.
  5. idle 상태로 전환된다. Title 필드가 비워지고 입력 모드로 복귀한다.
- 앱이 새벽 6시에 실행 중이지 않았더라도, 다음 번 앱 실행(또는 슬립에서 깨어남) 시 마지막으로 저장된 하루가 현재 하루와 다르면(즉 새벽 6시를 넘겼으면) 자동 마감을 수행한다.
  - 이때 종료 시각은 실제 새벽 6시로 기록하고, 진행 중이던 세션의 누적 시간은 그대로 보존하여 마감 하루에 포함한다.
- 여러 번의 새벽 6시를 건너뛴 경우(며칠째 앱을 켜지 않은 경우), 마지막 저장 상태를 한 번만 마감하고 새로운 하루로 진입한다(중간의 빈 날은 생성하지 않는다).

## 6. Title 규칙

### 6.1 입력

- Title 입력은 항상 선택사항이다. 비어 있어도 Start / Pause / Resume / End 모두 가능하다.
- 진행 중 세션의 Title은 언제든 수정 가능하다(시작 전, 중, 일시정지 중 모두).
- 입력은 한 줄, 자유 텍스트. 길이 제한은 두지 않지만 통계 화면에서는 너무 길면 잘려 보일 수 있다.

### 6.2 저장 표기

- Title이 빈 채로 세션이 종료되어 기록될 때, 저장 및 통계 표시상의 Title은 `(Untitled)`로 한다.
- 좌우 공백만 입력된 경우(`"   "`)도 빈 Title로 간주하여 `(Untitled)`로 저장한다.

### 6.3 정규화 규칙

- Title의 좌우 공백은 저장 전에 제거(trim)한다.
- 대소문자는 구분한다 (`React` 와 `react`는 다른 Title로 본다).
- 빈 문자열이나 공백만 입력된 Title은 `(Untitled)`로 정규화하여 저장된다.

## 7. 통계 규칙

### 7.1 무엇이 기록되는가

- 각 하루에는 그 하루 안에서 종료된 모든 세션이 시간 순서대로 기록된다.
- 세션 한 건당 기록되는 정보: Title, 누적 시간, 시작 시각.
- 누적 시간이 0인 세션(시작도 안 했거나, 시작 직후 즉시 리셋된 경우)은 기록하지 않는다.

### 7.2 표시 방식

- 하루를 통계 화면에서 보여줄 때, 세션을 시작 시각 오름차순으로 개별 항목으로 나열한다.
- 통계 화면 오늘 카드에서는 각 항목 옆에 오늘 전체 시간 대비 백분율을 함께 표시한다.
- 예시:
  ```
  리액트 공부      58%   2h 30m
  코드 리뷰        25%   1h 05m
  (Untitled)      17%   0h 45m
  ```

### 7.3 일평균 계산

- 일평균은 다음으로 계산한다.
  - 대상: 오늘을 제외한, 마감된 최근 7일.
  - 평균 = (대상 날들의 총 누적 시간 합) ÷ (대상 날 수).

## 8. 슬립 및 잠금 처리

### 8.1 자동 일시정지

- 다음 시점에 스탑워치가 실행 중이라면 자동으로 일시정지 상태로 전환된다.
  - 시스템이 슬립(잠자기) 모드로 진입하는 시점.
- 일시정지 시점의 Current와 Today는 그대로 보존된다.
- 자동 일시정지로 인해 손실되는 시간이 통계에 추가되지 않는다(자리 비운 시간을 작업 시간으로 치지 않는다).

### 8.2 자동 재개하지 않음

- 시스템이 깨어난 후 스탑워치를 **자동으로 다시 시작하지 않는다.**
- 사용자가 직접 Popover를 열고 `Resume` 버튼을 눌러야 재개된다.
  (슬립으로 인한 자동 일시정지는 항상 activeSession이 존재하는 paused 상태이므로 `Resume`이 표시됨.)

## 9. 데이터 보존 정책

- 모든 데이터는 사용자의 기기에 로컬로만 저장된다. 네트워크 전송 없음.
- 앱을 종료했다가 다시 열어도 다음 상태가 복원된다.
  - 오늘 하루와 그 하루 안에서 마감된 세션들.
  - 진행 중이던 세션의 Title과 누적 시간.
  - 종료 시점에 타이머가 실행 중이었더라도 복원 시에는 **항상 정지 상태로 시작한다**(슬립 처리와 같은 원칙: 자리 비운 시간을 가산하지 않는다).
    - 진행 중이던 세션이 있으면 paused 상태(`Resume` + `End` 버튼).
    - 진행 중이던 세션이 없으면 idle 상태(`Start` 버튼).
- 과거 날 데이터는 사용자가 명시적으로 지우기 전까지 무기한 보관된다.
  (보관 기간 자동 제한 없음.)

## 11. 비-목표 (Out of Scope)

이번 버전에서는 다음 기능들을 다루지 않는다.

- 여러 기기 간 동기화, 클라우드 백업.
- 글로벌 키보드 단축키.
- macOS 로그인 시 자동 실행 옵션(사용자가 직접 실행한다).
- 인터랙티브 드릴다운 차트, 범례, 줌 등 고급 시각화 기능. 주간 바 차트와 월간 캘린더는 제공하나 세부 차트 인터랙션은 제외한다.
- 알림(작업 시간 도달, 휴식 권장 등) 기능.
- 작업 카테고리/태그/색상 등의 분류 체계(Title만으로 식별).
- 데이터 내보내기/가져오기, CSV 등 외부 형식 지원.
- 동시에 여러 세션을 병렬로 돌리는 기능.
